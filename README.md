# QuellGTA_WFP
用于在QuellGTA工具中调用WFP进行网络控制的独立程序

## 为什么需要WFP
众所周知的，QuellGTA工具当前提供网卡断网和基于WDF防火墙断网的两种断网方式

前者相当于全局断网，且断网重连速度有可感知的延迟

而后者依赖于Windows Defender的防火墙功能，一旦用户自行关闭防火墙功能或安装了其他自带防火墙的杀毒软件将其替换，则不生效

恰恰在许多网吧环境，或是对设备性能、安全性有追求的用户身上往往Windows Defender的防火墙功能是被关闭的

因此，我们可以采用更现代也更底层的Windows Filter Platform进行网络控制，既保证了兼容性，又拥有和WDF防火墙断网相当的体验

## 为什么是独立程序
必须承认，这是我技术栈上的缺陷

起初，我试图通过易语言支持库的方式进行补充，但无奈，VC98链接器过于古老，我编译的静态库在CRT方面遇到了一些难以解决的问题

后来我尝试使用DLL的方式进行补充，其次看起来是个不错的做法，毕竟易语言本身加载DLL的方式就是延迟加载，但不知为何导出表中分明就有的导出函数，却无法被识别，遂放弃

<img width="1321" height="510" alt="beb5914fefade35e404a7107de80e190" src="https://github.com/user-attachments/assets/c45305da-74c7-4147-915c-523b671249e7" />

而QuellGTA工具本身就通过携带devcon二进制文件的方式实现devcon控制网卡开关，因此我认为再多携带一份二进制文件不是太大的问题。

当然，虽然设计上是针对QuellGTA工具调用而编写的，实际上也可以独立运行，或者也可以通过相同的方式调用，希望能给其他开发者一些帮助吧。
